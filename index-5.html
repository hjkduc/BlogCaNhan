Làm việc với địa chỉ mạng trong Java

Trước khi tạo ra các kết nối, một ứng dụng mạng cần biết "nói chuyện" với ai. Điều này được thực hiện thông qua các địa chỉ mạng. Java cung cấp lớp InetAddress để làm việc với cả địa chỉ IP (IPv4 và IPv6) và tên miền (hostname).

Lớp InetAddress là gì?

Lớp InetAddress không có hàm khởi tạo (constructor) công khai. Thay vào đó, chúng ta sử dụng các phương thức tĩnh (static methods) để tạo ra các đối tượng của nó. Điều này giúp quản lý và phân giải tên miền một cách hiệu quả hơn.

Các phương thức phổ biến

  • getByName(String host): Phương thức quan trọng nhất. Nó nhận vào một tên miền (ví dụ: "google.com") hoặc một chuỗi địa chỉ IP và trả về một đối tượng InetAddress. Quá trình này được gọi là phân giải DNS.
  • getLocalHost(): Trả về địa chỉ của máy tính cục bộ (localhost).
  • getAllByName(String host): Trả về một mảng các địa chỉ IP nếu một tên miền trỏ đến nhiều máy chủ.
  • getHostName() / getHostAddress(): Sau khi có đối tượng InetAddress, hai phương thức này giúp lấy ra tên miền và địa chỉ IP tương ứng.

Ví dụ thực tế


import java.net.InetAddress;
import java.net.UnknownHostException;

public class InetAddressExample {
    public static void main(String[] args) {
        try {
            // Phân giải tên miền google.com
            InetAddress address = InetAddress.getByName("google.com");
            System.out.println("Hostname: " + address.getHostName());
            System.out.println("IP Address: " + address.getHostAddress());

            // Lấy thông tin máy local
            InetAddress localAddress = InetAddress.getLocalHost();
            System.out.println("Local Hostname: " + localAddress.getHostName());
            System.out.println("Local IP Address: " + localAddress.getHostAddress());

        } catch (UnknownHostException e) {
            e.printStackTrace();
        }
    }
}

Việc sử dụng thành thạo lớp InetAddress là bước đầu tiên và cơ bản nhất để xác định và tương tác với các thiết bị khác trên mạng.
