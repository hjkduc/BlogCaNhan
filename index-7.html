Lập trình Socket UDP: Khi tốc độ là ưu tiên hàng đầu

Giao thức UDP (User Datagram Protocol) hoàn toàn trái ngược với TCP. UDP là một giao thức phi kết nối (connectionless) và không đáng tin cậy. Nó hoạt động theo triết lý "gửi và quên" (fire and forget). Khi bạn gửi một gói tin UDP (Datagram), bạn không biết liệu nó có đến đích hay không, có bị lặp lại hay đến sai thứ tự hay không.

1. So sánh TCP và UDP: Quyết định thiết kế

Việc lựa chọn giữa TCP và UDP là một quyết định thiết kế quan trọng, dựa trên sự cân bằng giữa tốc độ và độ tin cậy:

Đặc điểm TCP (Transmission Control Protocol) UDP (User Datagram Protocol)
Mô hình Hướng kết nối (Cần bắt tay 3 bước) Phi kết nối (Không cần bắt tay)
Độ tin cậy Cao (Đảm bảo dữ liệu đến, không mất, đúng thứ tự) Thấp (Không đảm bảo)
Tốc độ Chậm hơn (Do cần ACK và kiểm soát luồng) Nhanh hơn (Ít overhead)
Ứng dụng HTTP, FTP, Email (cần chính xác) Game online, Video/Voice Streaming, DNS (cần tốc độ)

2. Các lớp Java cho Socket UDP

Lập trình UDP trong Java xoay quanh hai lớp chính:

  • `DatagramSocket`: Là Socket được sử dụng để gửi và nhận các gói tin Datagram. Nó không cần thiết lập kết nối.
  • `DatagramPacket`: Là gói tin thực tế được gửi qua mạng. Mỗi gói tin chứa dữ liệu (byte array), địa chỉ IP đích, và Port đích.

3. Code chi tiết: Gửi và nhận gói tin

Server.java (Receiver):


// UDPServer.java
import java.net.*;
public class UDPServer {
    public static void main(String[] args) throws Exception {
        DatagramSocket serverSocket = new DatagramSocket(9876);
        System.out.println("UDP Server is running...");
        byte[] receiveData = new byte[1024];
        while(true) {
            DatagramPacket receivePacket = new DatagramPacket(receiveData, receiveData.length);
            serverSocket.receive(receivePacket);
            String sentence = new String(receivePacket.getData());
            System.out.println("RECEIVED: " + sentence);
        }
    }
}

Client.java (Sender):


// UDPClient.java
import java.net.*;
public class UDPClient {
    public static void main(String[] args) throws Exception {
        DatagramSocket clientSocket = new DatagramSocket();
        InetAddress IPAddress = InetAddress.getByName("localhost");
        byte[] sendData = "Hello from UDP Client!".getBytes();
        DatagramPacket sendPacket = new DatagramPacket(sendData, sendData.length, IPAddress, 9876);
        clientSocket.send(sendPacket);
        clientSocket.close();
    }
}

UDP là minh chứng cho việc không có giải pháp nào tốt nhất cho mọi vấn đề. Sự đánh đổi giữa tốc độ và độ tin cậy là một quyết định thiết kế quan trọng.
